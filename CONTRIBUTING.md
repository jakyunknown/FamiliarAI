## 🚀 Getting Started

1. **Fork** the repository on GitHub
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/YOUR-USERNAME/FamiliarAI.git
   ```
3. **Install** dependencies:
   ```bash
   npm install
   ```
4. **Create a branch** for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```
5. **Start** the development server:
   ```bash
   npm run dev
   ```

## 🤝 How to Contribute

There are many ways to contribute to Familiar AI:

- 🐛 Fix bugs
- ✨ Add new features
- 📖 Improve documentation
- 🎨 Improve UI/UX
- 🧪 Write tests
- 💡 Suggest ideas via GitHub Issues

---

## 🔄 Pull Request Process

1. Make sure your code runs without errors
2. Update the README if your changes affect how the project works
3. Keep pull requests focused — one feature or fix per PR
4. Write a clear PR description explaining what you changed and why
5. Link any related GitHub Issues in your PR description
6. Wait for a review from one of the maintainers:
   - Judah Coskun
   - Pranav Emmadi
   - Ronak Rupani

---

## 🎨 Code Style

- Use **TypeScript** for all new files
- Follow the existing folder structure under `src/`
- Keep components small and reusable
- Use Tailwind utility classes for styling — avoid custom CSS where possible
- Use descriptive variable and component names

---

## 🐛 Reporting Bugs

Found a bug? Please open a GitHub Issue and include:

- A clear description of the bug
- Steps to reproduce it
- What you expected to happen vs. what actually happened
- Screenshots if applicable
- Your browser and OS

---

## 💡 Suggesting Features

Have an idea? Open a GitHub Issue with:

- A clear description of the feature
- Why it would be useful for caregivers or patients
- Any mockups or examples if you have them

---

## 🤲 Code of Conduct

This project is built to help vulnerable people. We ask that all contributors:

- Be respectful and inclusive
- Be mindful that this tool impacts dementia patients and their families
- Give constructive feedback
- Support and encourage new contributors
